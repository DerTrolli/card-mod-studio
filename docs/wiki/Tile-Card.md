`type: tile` — full support; the tile is one of the best-covered cards.

## Available options

| Module | Available | Notes on this card |
|---|---|---|
| Font | ✅ | Name + state text, via the tile's own `--ha-tile-info-*` variables (handled automatically) |
| Visual Filters | ✅ | |
| Accent Color | ✅ | Sets `--tile-color` — the icon + state tint. **Tile features follow too** (bar gauge, toggle rows derive their color from it) |
| Icon Color | ✅ | Overrides just the icon, independent of the tile color |
| Threshold Colors | ✅ | All properties; *Accent Color* is usually the one you want here |
| Background | ✅ | |
| Animation | ✅ | |
| Border & Radius | ✅ | |
| Advanced CSS | ✅ | |

## Card-specific behavior

- **Why accent needs the Studio:** HA computes the tile's color from entity
  state and writes it as an *inline style* on every render — a naive CSS
  variable silently loses. The Studio emits `--tile-color: … !important`,
  which wins reliably (this was one of the v0.7.1 headline fixes).
- **Tile features** (bar gauge, toggles, controls under the tile) inherit
  the accent through `--feature-color` — a threshold-driven accent recolors
  the bar gauge live.
- **State awareness** follows the tile's entity: toggleable domains get the
  ON/OFF modes directly; sensors etc. can still use "Controlled by".

## Example

```yaml
type: tile
entity: sensor.outside_temperature
card_mod:
  style: |
    ha-card {
      --accent-color: {{ '#f44336' if states('sensor.outside_temperature') | float(0) >= 25 else '#2196f3' }};
      --tile-color: {{ '#f44336' if states('sensor.outside_temperature') | float(0) >= 25 else '#2196f3' }} !important;
    }
```
*(Generated by Threshold Colors → Accent Color with two rules.)*
